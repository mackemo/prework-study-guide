HTML

  • Hypertext Markup Language
  • head element: metadata, non-visible info of webpage.
  • body element: info shown to the user.
  • title is important for SEO (Search Engine OPtimization).
  • UTF-8: char set with all letters and symbols.
  • h1-6 heading sizes, p paragraph, ul unordered, ol ordered, li list items
  • img info about images, br empty line or line break, a anchor and link to webpage

CSS

  • Cascading Style Sheets
  • inline: in html tag, less flexible, internal: embed entire sheet, external: separate file (most common)
  • separation of concerns - code in diff files based on use
  • selector {property: value;} declaration
  • * : wild card selector (for all elements)
  • link external CSS to html file in metadata

Git

  • Git: Version Control
  • git clone tag copied ssh url tag: adds to local computer
  • git status: see current branch you're on
  • git checkout -b 'name of branch': creates new branch
  • git add -A: staging/saves changes
  • git commit -m "message": detailed commit message
  • git pull origin main: check for changes are up to date
  • git push origin 'name of branch': allows changes on branch
  • git checkout main: switch to main branch

JavaScript

  • Adds logic and interactivity
  • variables: const and let
  • const: variables cant be reassigned, let: variables can
  • to access or execute: console.log(name) (.log is method or action, name is arguement or output)
  • tag script tag attaches .js to html (under footer in body)
  • Chrome DevTools: for developers to inspect, edit, debug
  • Control Flow: order of execution
  • controlling flow with functions, conditionals, arrays, and loops
  • Conditional: if statements
  • Array: single variable with group of data, index: location of data in array
  • For loops: repeats code for certain # of times
  • commenting out code by using /* code */: makes code inactive without deleting it
  • Functions: instructions for computer to perform task (only executed when called)
  • first define then call a function
